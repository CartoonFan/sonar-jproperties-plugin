Defining several times the same key across files may be error prone. Thus, you should carefully review those cross-file duplicated keys and merge them whenever possible.

This rule logs an issue each time a duplicated key is encountered. For instance, if a key is defined in 3 different files, 3 issues are raised.

To prevent some files from being check for cross-file duplicated keys (i18n files for example), set exclusions. For instance, to exclude all files in any i18n directory, set the Ignore Issues on Multiple Criteria property to:

  • Rule key pattern: jproperties:duplicated-keys-across-files
  • File path pattern: **/i18n/**
