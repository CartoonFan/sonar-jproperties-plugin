For readability reasons, the same separator should be used all the time.

Rule can be configured to check one of the following formats:

  • key=element
  • key:*whitespace*element

Noncompliant Code Example

With format "key:*whitespace*element": 
key1=value1
key2 : value2
key3 :value3
key4:    value4

Compliant Solution

With format "key:*whitespace*element": 
key1: value1
key2: value2
key3: value3
key4: value4
